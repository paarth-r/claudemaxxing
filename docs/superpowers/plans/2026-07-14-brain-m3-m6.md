# brain M3-M6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make rules agent-authored instead of hand-written, keep the brain's context cost near zero, and ship the whole thing as something a stranger can install.

**Architecture:** Four capture paths write into `.brain/` with no user involvement: `/brain init` mines rules out of docs the repo already has; a `UserPromptSubmit` hook queues corrections; the `PostToolUseFailure` hook queues pain; a `SessionEnd` distiller turns the queues plus the transcript into rules and notes via a headless `claude --print` call. A lifecycle module promotes rules that earn their keep and archives the ones that do not. An index generator keeps the always-loaded footprint at ~200 tokens. Vault mirroring is an optional configured export, off by default.

**Tech Stack:** Python 3.9 stdlib, pytest, Claude Code plugin hooks + skills, headless `claude --print`.

## Global Constraints

Everything from the M1-M2 plan still holds, and these are load-bearing:

- **Python 3.9 floor.** Bare `python3` is Xcode's 3.9.6. Every module starts with `from __future__ import annotations`. No `X | Y` unions at runtime.
- **Zero third-party dependencies.** Stdlib only.
- **Fail open, always.** Any exception, parse error, or timeout exits 0 and allows the action. Exit code 2 is never used.
- **Zero footprint without `.brain/`.** Every hook's first action is discovery; absent, exit immediately and print nothing.
- **Nothing personal is hardcoded.** No vault path, no username, no `secondbrain`, no Obsidian assumption anywhere in the code. The vault mirror is a configured path, off by default. This is the difference between a tool other people can install and a personal script.
- **`settings.json` is never written to.**
- **No emojis.** Single-line shell commands only.
- **README ships in the same commit as the feature.**

## The installability question, settled

The core is already universal. The only personal thing in the whole design is the Obsidian
vault, so it becomes configuration rather than architecture — resolved in three layers:

```
1. .brain/config.yml     (this repo)         highest priority
2. ~/.brain/config.yml   (this machine)      the user-level default
3. built-in defaults     (mirror: off)       what a stranger gets
```

```yaml
# ~/.brain/config.yml  -- personal, never shipped, never in the repo
mirror: ~/Library/.../secondbrain/projects
```

```yaml
# .brain/config.yml  -- per repo, committed only if you choose
paused: false
auto_remedy: true
# mirror: inherited from the user-level config unless overridden here
```

**Off by default in the box, on by default for the author.** A stranger who clones this
never creates `~/.brain/config.yml`, so `mirror` is unset and the brain never writes a
single byte outside their repo — Obsidian is never even mentioned. Setting the machine-level
config once turns mirroring on for every repo automatically, with no per-repo setup.

This is the same layering the kill switch already uses (`~/.brain/DISABLED`), so it is a
pattern the codebase has, not a new concept. Nothing personal is hardcoded anywhere; the
personal knowledgebase becomes one *configuration* of a general feature. No fork, no
personal-use-only compromise.

## MILESTONE 3 — Capture

### Task 10: Rule lifecycle (SHIPS BEFORE ANY AGENT-AUTHORED RULE)

The safety valve. The moment a model writes rules, bad rules appear that no human
approved, and they must decay and die on their own evidence.

**Files:** Create `hookkit/lifecycle.py`, `tests/brain/test_lifecycle.py`. Modify `scripts/guard.py`.

**Interfaces:**
- Produces `hookkit.lifecycle.review(rule) -> str` returning one of `"keep"`,
  `"promote"`, `"demote"`, `"archive"`, and `hookkit.lifecycle.apply(rule, brain) -> None`
  which rewrites `severity:` in place or moves the file to `_archive/` with a reason
  appended to the body.

Thresholds (deliberately conservative, because a wrongly-archived rule is cheap to
rewrite and a wrongly-enforced rule is expensive to live with):

| Condition | Action |
|---|---|
| `overridden >= 3` | **archive** — it keeps being wrong |
| `severity: block` and `overridden >= 1` | **demote** to `warn` — it lost its promotion |
| `severity: warn`, `satisfied >= 5`, `overridden == 0` | **promote** to `block` — it earned it |
| otherwise | keep |

- [ ] **Step 1:** Write `tests/brain/test_lifecycle.py` covering: a rule with 3 overrides archives; a `block` rule with 1 override demotes to `warn`; a `warn` rule with 5 satisfies and 0 overrides promotes to `block`; a fresh rule is kept; archiving moves the file into `_archive/` and leaves `rules/` clean; archiving appends a human-readable reason to the body; `apply` on an unwritable path is a silent no-op (fail open).
- [ ] **Step 2:** Run `python3 -m pytest tests/brain/test_lifecycle.py -v`. Expected: FAIL, no module.
- [ ] **Step 3:** Implement `hookkit/lifecycle.py`. `review` is pure (takes a `Rule`, returns a verdict string). `apply` does the I/O and swallows every `OSError`.
- [ ] **Step 4:** Wire into `scripts/guard.py`: after each `stats.bump`, re-load that rule and call `lifecycle.apply`. A rule that just took its third override archives itself immediately.
- [ ] **Step 5:** Run `python3 -m pytest tests/brain -q`. Expected: all pass.
- [ ] **Step 6:** Commit: `feat(brain): rules promote, demote, and archive on their own evidence`

### Task 11: Capture queues and correction detection

**Files:** Create `hookkit/queue.py`, `hookkit/correction.py`, `scripts/capture.py`, `tests/brain/test_queue.py`, `tests/brain/test_correction.py`. Modify `scripts/receipt.py`, `hooks/hooks.json`.

**Interfaces:**
- `hookkit.queue.push(brain, kind, payload: dict) -> None` — append to `.brain/_queue/<kind>.jsonl`
- `hookkit.queue.drain(brain, kind) -> list[dict]` — read all and truncate the file
- `hookkit.queue.peek(brain, kind) -> list[dict]` — read without truncating
- `hookkit.correction.looks_like_correction(prompt: str) -> bool`

`looks_like_correction` is a cheap heuristic, not a judgement — it runs on every prompt,
so it must be free. It flags second-person imperatives and prohibitions: `no,` / `don't` /
`never` / `always` / `you need to` / `you have to` / `stop` / `make sure you` / `remember to`.
False positives are fine and expected: the distiller (a real model) decides what is
actually a rule. False negatives are the only real cost.

### Task 12: The distiller

**Files:** Create `hookkit/distiller.py`, `scripts/distill.py`, `tests/brain/test_distiller.py`. Modify `hooks/hooks.json`.

The distiller is the only component that calls a model. It runs once, at `SessionEnd`, on
Sonnet, and only on substantial sessions.

**Interfaces:**
- `hookkit.distiller.build_prompt(brain, corrections, pain, transcript_excerpt, existing_rule_ids) -> str`
- `hookkit.distiller.parse_response(text) -> list[dict]` — the model replies with a JSON
  array of file operations: `[{"path": "rules/x.md", "content": "..."}]`
- `hookkit.distiller.apply(brain, ops) -> list[str]` — write the files, return what changed
- `hookkit.distiller.run(brain, session_id, transcript_path) -> list[str]` — the whole thing

Design rules that matter:
- **The model is told the exact rule schema** (flat dotted keys) and given the existing
  rule ids so it updates rather than duplicates.
- **New rules are always born `severity: warn`.** The prompt says so, and `apply` rewrites
  `severity:` to `warn` regardless of what the model emitted. Never trust the model to set
  its own enforcement level.
- **`apply` only writes under `.brain/`.** Any path containing `..` or starting with `/`
  is rejected. A model-authored path is untrusted input.
- **A malformed response writes nothing.** The session simply ends without learning.
- Queues are drained only on a successful apply, so a failed distillation is retried next
  session rather than lost.

- [ ] **Step 1:** Write `tests/brain/test_distiller.py`: `parse_response` handles a bare JSON array, a fenced ```json block, and trailing prose; returns `[]` for garbage rather than raising; `apply` writes files under `.brain/`; **`apply` REJECTS `../../etc/passwd` and `/etc/passwd` and any absolute path**; `apply` forces `severity: warn` even when the model emitted `severity: block`; `build_prompt` includes the existing rule ids and the flat-key schema. Do NOT call a real model in tests — `run` takes an injectable `call_model` function.
- [ ] **Step 2:** Run. Expected: FAIL.
- [ ] **Step 3:** Implement `hookkit/distiller.py`. `run(brain, session_id, transcript_path, call_model=_claude)` where `_claude` shells out to `claude --dangerously-skip-permissions --print` with the prompt on stdin, `timeout=180`, and returns stdout. Any failure returns `[]`.
- [ ] **Step 4:** Implement `scripts/distill.py` (`SessionEnd`): discovery, killswitch, skip if fewer than 3 user turns in the transcript, call `distiller.run`, then `index.generate`, then `mirror.export` if configured. Log what it wrote to `.brain/_log/distill.log`. Print nothing.
- [ ] **Step 5:** Register `SessionEnd` in `hooks/hooks.json` with `timeout: 300`. **Verify it does not conflict with the user's two existing `SessionEnd` hooks** — plugin hooks and settings hooks coexist; both should run.
- [ ] **Step 6:** Run `python3 -m pytest tests/brain -q`.
- [ ] **Step 7:** Commit: `feat(brain): SessionEnd distiller turns corrections and pain into rules`

### Task 13: Index generation

**Files:** Create `hookkit/index.py`, `tests/brain/test_index.py`. Modify `scripts/context.py` (it already reads `index.md`; nothing writes it yet).

**Interfaces:**
- `hookkit.index.generate(brain) -> str` — build the index text and write `.brain/index.md`

The index is the entire always-loaded context cost of the brain, so it is budgeted, not
merely short. Every note contributes exactly one line: its path and its `summary:`
frontmatter field (or its first heading if absent). Rules contribute **a count, not
lines** — they are enforced by hooks and cost nothing to *not* know.

### Task 14: Layered config, then the mirror

**Files:** Create `hookkit/config.py`, `hookkit/mirror.py`, `tests/brain/test_config.py`, `tests/brain/test_mirror.py`. Modify `hookkit/killswitch.py` to delegate to `config`.

**Interfaces:**
- `hookkit.config.get(brain, key, default=None) -> str | bool | int | None` — resolves
  `.brain/config.yml`, then `~/.brain/config.yml`, then the passed default.
- `hookkit.config.flag(brain, key, default: bool) -> bool` — the boolean form.
- `hookkit.mirror.target(brain) -> Path | None` — `config.get(brain, "mirror")`, `~`-expanded.
- `hookkit.mirror.export(brain, repo_name) -> int` — copy `.brain/` (minus `_receipts/`,
  `_queue/`, `_log/`) into `<target>/<repo_name>/`; returns the file count.

The layering is what makes this both shippable and personally useful: **off by default in
the box, on by default for the author**, with nothing personal in the code.

One-way, always. A copy, never a symlink: iCloud Drive mangles symlinks, and a vault
symlinked into a repo that later gets `rm -rf`'d leaves dead links behind.

## What is deliberately still not built

- **Rule conflict detection.** Two rules that gate the same command with contradictory
  remedies are not detected. The lifecycle handles it crudely: the wrong one accumulates
  overrides and archives itself.
- **Cross-repo rule sharing.** Each brain is an island. Deliberate: it is what keeps a
  Mashgin rule out of an FRC repo.
- **Rule editing UI.** Rules are files. Edit them in Obsidian or a text editor.
